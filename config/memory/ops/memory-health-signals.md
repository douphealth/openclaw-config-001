# Memory Health Signals

## Healthy Signs
- Memory search enabled with indexed files/chunks
- Semantic search returns useful, non-repetitive results
- Active sites/projects have current entity files
- Daily notes are chronological, not duplicated current state
- MEMORY.md contains durable truths, not raw logs

## Degraded Signs
- Memory enabled but index still empty
- Repeated blockers scattered across many daily files
- Entity files missing for active work
- Semantic search returns repetitive or stale snippets
- MEMORY.md contains superseded or temporary facts

## Blocked Signs
- Memory search disabled
- Embedding backend unavailable
- Indexing repeatedly interrupted
- Session memory expected but absent

## Fix Mapping
| Signal | Fix |
|--------|-----|
| Technical blocker | Run memory status/index, fix config/provider/runtime |
| Structure blocker | Create/update entity files |
| Drift blocker | Run memory review, consolidate current truth |
