# Weekly Memory Review Checklist

## Goal
Keep memory high-signal, retrievable, and operationally useful.

## Steps
1. Read the last 7 daily memory files
2. Identify repeated facts or repeated blockers
3. Promote durable current-state facts into entity memory first:
   - Relevant `memory/sites/*.md`
   - Relevant `memory/projects/*.md`
   - Relevant `memory/ops/*.md`
   - Relevant `memory/people/*.md`
4. Promote only stable, cross-session truths into MEMORY.md
5. Remove or consolidate noisy duplication
6. Mark stale or resolved blockers as resolved
7. Update next actions in active site/project files
8. Confirm long-term memory still reflects reality

## Promotion Test
Promote only if the item is:
- Likely to matter in future sessions
- Tied to a real site/project/person/system
- Not just transient chat context
- Actionable or strategically durable

## Noise Test
Do NOT preserve if it is:
- Repetitive
- Obvious from other files
- Emotionally charged but operationally empty
- Already superseded by a newer fact
