# Memory Maintenance System

## Purpose
Make memory quality a maintained system, not a one-time cleanup.

## Cadence

### Daily
- Write meaningful notes to `memory/YYYY-MM-DD.md`
- Update entity files when current state changes materially

### Weekly
- Run the memory review macro
- Consolidate duplicates
- Update current truths in entity files first
- Promote only durable cross-session lessons into MEMORY.md

### Periodic
- Prune stale assumptions
- Verify active site/project files match reality
- Ensure MEMORY.md contains durable truths, not raw history

## Retrieval Discipline
Use this order:
1. Entity file
2. Semantic memory search
3. Recent daily notes
4. MEMORY.md

## Signals of Memory Drift
- Same blocker repeated across many days
- Old truths outranking current ones
- Entity files no longer matching reality
- MEMORY.md becoming a dumping ground
- Retrieval returning repetitive snippets

## Fixes for Drift
- Consolidate repeated facts into one current bullet
- Update entity files directly
- Prune stale notes from long-term memory
- Use daily notes for chronology, not as final resting place
