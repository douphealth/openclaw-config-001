# Semantic Keyword Integration (Natural, Non-Spam)

## Goal
Make every review substantially more useful, complete, and context-rich by naturally covering the topic graph around the primary query.

## Rules
1. Start with user intent, not keyword density.
2. Use semantic entities/phrases where they clarify decisions.
3. Avoid forced repetition of exact-match terms.
4. Cover related subtopics only when relevant to buyer decisions.
5. Prefer natural language variants over robotic keyword stuffing.

## Required Coverage Buckets (per review)
- Core product entity (exact model + category)
- Buyer intent terms (best for, worth it, alternatives, who should buy)
- Performance entities (battery, accuracy, reliability, usability, value)
- Comparison entities (vs alternatives, trade-offs)
- Risk/limitation entities (known constraints, uncertainty)
- Decision entities (buy/test/skip logic)

## Practical Pattern
- Intro: 1–2 natural mentions of core entity + buyer intent
- Core sections: semantically related terms per section goal
- FAQ: entity-rich question variants users actually ask
- Verdict: summarize with context-specific recommendation language

## Anti-Patterns (auto-fail)
- Exact-match term repeated unnaturally in consecutive paragraphs
- Lists of keywords pasted without reader value
- Sections added only to include a keyword
- Generic “SEO paragraph” not tied to review evidence
