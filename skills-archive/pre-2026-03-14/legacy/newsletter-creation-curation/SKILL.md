# Newsletter Creation & Curation OS

Execute in order:
1) Audience + objective definition
2) Content sourcing + curation shortlist
3) Issue drafting
4) QA + compliance
5) Scheduling + automation
6) Performance review loop

## 1) Audience + objective definition
- Define newsletter goal: nurture, conversion, retention, authority.
- Define segment: new subscribers, active readers, dormant, buyers.
- Set KPI target per issue: open rate, CTR, unsubscribe ceiling.

## 2) Content sourcing + curation shortlist
- Collect fresh inputs: internal posts, product updates, trusted external sources.
- Score each candidate by relevance, novelty, actionability, and trust.
- Keep only high-signal items; avoid filler.

## 3) Issue drafting
- Build issue structure:
  - Subject line variants (3)
  - Preview text variants (2)
  - Lead hook
  - 3–7 curated blocks with clear transitions
  - Single primary CTA (+ optional secondary CTA)
- Maintain concise, human tone and strong scannability.

## 4) QA + compliance
- Verify every claim/link before finalizing.
- Enforce consent and unsubscribe visibility.
- Remove spam-trigger phrasing and deceptive urgency.

## 5) Scheduling + automation
- Choose send window by audience timezone + historical engagement.
- Route to the correct automation path (welcome, nurture, promo, re-engagement).
- Use reversible rollout: test segment first, then full send.

## 6) Performance review loop
- Compare actual vs KPI target after send.
- Record winners/losers for subject, hook, CTA, content type.
- Feed findings into next issue brief.

## Output artifact (required)
- audience_segment
- objective
- issue_outline
- final_copy
- subject_preview_variants
- CTA_map
- schedule
- KPI_target
- post-send findings (when available)

## Safety gates
- No fabricated metrics or claims.
- No purchased-list or non-consented recipient usage.
- No “done” status without sent/test evidence.
