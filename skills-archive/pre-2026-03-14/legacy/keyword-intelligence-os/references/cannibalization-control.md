# Cannibalization Control

## Detect
- Multiple URLs targeting near-identical query intent.
- Same primary entity + same intent + overlapping headings.

## Resolve
1. Keep strongest URL as canonical target.
2. Re-scope weaker page to adjacent sub-intent.
3. Rewire internal links to winner URL for primary anchor family.
4. Use canonical only when merger/re-scope is not feasible.

## Validation
- Exactly one primary URL per keyword family.
- Supporting URLs use differentiated modifiers and headings.
