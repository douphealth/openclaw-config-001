# WordPress SOTA Content Template (A-M)

## A) Technical SEO metadata
- **Title tag (55-65 chars):** PRIMARY_KEYWORD + year + factual modifier.
- **Meta description (145-160 chars):** keyword + concrete value + CTA.
- **Slug:** short, lowercase, hyphenated keyword.

## B) Answer-first snippet paragraph (40-60 words)
Directly answer query in sentence 1. Name 2-3 entities and 1 source-backed stat.

## C) Quick answer callout
2-3 concise sentences. Specific, citable.

## D) TOC
Anchor list of key sections.

## E) Introduction (200-300 words)
Include:
- specific first-hand statement
- reader pain point
- what is covered (bullet list)
- one sourced stat

## F) Definition section (150-200 words)
Simple definition + real-world example + sub-entities.

## G) Evaluation criteria (4-7 H3 criteria)
Each criterion: why it matters + proof signal.

## H) Comparison table
For list/best/comparison intents:
- >=8 rows
- >=5 columns
- include Best For and Not Ideal For signals

## I) Deep-dive sections
Per item/entity:
- one-sentence verdict
- key specs bullets
- practical context
- best for / not ideal for

## J) Expert experience section (200-400 words)
Failures, wins, and lessons from direct use.

## K) Common mistakes
4-6 pitfalls with fixes.

## L) FAQ section
6-10 questions, each answer 40-80 words.

## M) Conclusion + CTA
One-line recommendation + 3-step action plan + CTA.

---

## Trust blocks to include
- **Editorial standard:** documented methodology + update policy.
- **Affiliate disclosure:** compensation does not change scoring/safety guidance.
- **Last reviewed:** visible date.

## JSON-LD to include
- FAQPage
- Article
- BreadcrumbList (if route known)
- HowTo (only if article is procedural)
