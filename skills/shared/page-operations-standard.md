# Page Operations Standard

Use this standard for meaningful page/post changes, especially when the page affects SEO, UX, monetization, or trust.

## Applies to
- homepage work
- money pages
- fragile custom pages
- review pages
- comparison pages
- visually broken pages
- page rewrites or shell repairs

## Core flow
1. Verify access and current state
2. Back up the page/post before modification
3. If visuals matter, capture baseline desktop/mobile screenshots
4. If mobile is broken, run overflow diagnostics first
5. Make the smallest structural fix first
6. If repeated regressions persist, escalate to controlled shell rewrite
7. Re-verify live output
8. For visual work, capture post-change desktop/mobile screenshots
9. Do not claim success without proof matching the task type

## Money-page minimums
- clear buyer-intent title/H1
- conversion-supportive internal links
- trust/disclosure/methodology support
- mobile-safe rendering
- decision-support structure, not just affiliate links

## Fragile-page minimums
- baseline screenshots before edits
- overflow diagnostics when relevant
- shell/container fixes before local override spam
- page-safe rewrite when patching is no longer efficient or safe

## Verification rule
- source checks for structure
- live fetch for publication/render presence
- screenshots for appearance/layout
- browser diagnostics for overflow/console/layout issues
