# SEO / AEO / GEO Superpowers Layer

## Purpose
Shared SOTA upgrade layer for SEO, content, GEO, AEO, entity, editorial, and competitor skills in this repo.

This layer merges the strongest reusable patterns into one safe standard:
- AEO prompt research
- AEO content engineering for citation-worthiness
- AI visibility analytics
- GEO scoring for AI citation readiness
- CORE-EEAT-lite quality control
- Safe SERP/browser research
- Optional Ahrefs enrichment when credentials and budget justify it

## Non-Negotiable Rules
1. Optimize for ranking + extractability + citation-worthiness + conversion intent.
2. Build a prompt universe, not just a keyword list.
3. Ground recommendations in observed evidence.
4. Prefer direct answers over generic prose.
5. Use fresh, attributed facts.
6. Treat AI visibility as measurable.
7. Do not adopt unsafe scraping or brittle automation as defaults.

## Universal Research Stack
### Layer 1 — Search Demand
Use GSC/Bing data, web search, and competitor evidence to capture:
- high-impression queries
- low-CTR opportunities
- page-2 opportunities
- SERP features
- high-intent modifiers

### Layer 2 — AI Prompt Universe
For each strategic topic, generate:
- problem-aware prompts
- solution-aware prompts
- comparison prompts
- best-of / alternatives prompts
- how-to prompts
- evaluation prompts
- persona/vertical prompts

Prompt priority formula:
`Priority = (Relevance × 2 + VolumePotential + Winability + IntentValue) / 5`

### Layer 3 — Citation Landscape
Inspect current winners for:
- direct-answer formatting
- definition blocks
- FAQ coverage
- comparison tables
- freshness
- named sources
- authority signals
- what they still miss

### Layer 4 — CORE-EEAT Lite
Score:
- Contextual clarity
- Organization
- Referenceability
- Exclusivity / unique value
- Experience
- Expertise
- Authoritativeness
- Trust

## GEO / AI Citation Dimensions (/40)
1. Definition clarity
2. Quotable statements
3. Factual density
4. Source attribution
5. Question coverage
6. Authority signals
7. Freshness
8. Structural extractability

Interpretation:
- 32–40 = AI-ready
- 24–31 = optimize further
- 16–23 = weak
- 0–15 = non-citable

## Mandatory On-Page Patterns
Where appropriate, include:
- definition paragraph near the top
- direct-answer blocks under key headings
- FAQ/Q&A sections
- comparison tables
- named stats with dates/sources
- author/reviewer/entity trust signals
- related-resource/internal-link blocks

## AI Visibility Measurement
Track:
- Mention Rate
- Citation Rate
- Primary Citation Rate
- AI Share of Voice
- Tier-1 Prompt Coverage

## Safe Browser / SERP Rules
- Prefer cheap/stable evidence paths first.
- Use browser automation only when JS or visual proof is required.
- Document observed evidence, not just attempted steps.
- Respect rate limits and avoid burst scraping.

## Optional Ahrefs Enrichment
Only when credentials already exist and cost is justified:
- minimize units
- batch where possible
- respect rate limits
- cache/reuse findings
- use as enrichment, not sole truth
