# SERP + AI Research Protocol

## Goal
Turn SERP research, fetched pages, competitor pages, and AI-answer observations into reliable action.

## Workflow
1. Demand capture — GSC/Bing first, then live SERP framing.
2. Query expansion — SEO keywords + natural-language AI prompts + comparison/evaluation variants.
3. Result harvesting — capture content format, answer style, source quality, freshness, entities, schema, and structure.
4. Gap detection — find where current winners are outdated, generic, weakly sourced, or poorly extractable.
5. Action synthesis — new page, rewrite, FAQ block, definition block, comparison table, schema, trust signals, CTR fix, internal links.

## Safe Evidence Rules
- Prefer 3-5 strong sources over 20 weak ones.
- Record source URLs/domains for strategic recommendations.
- Separate observed facts from inferred opportunities.
- If a conclusion depends on dynamic/browser state, note that explicitly.
