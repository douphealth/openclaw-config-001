# Skill Efficiency Standard

## Purpose
Make skills execute with less waste, less drift, and higher professional quality.

## Core Rules
1. Route to the most specific useful skill quickly.
2. Prefer the smallest decisive operation first.
3. Use fewer, stronger writes instead of noisy micro-edits.
4. Prefer structured outputs over rambling prose.
5. Keep execution resumable when work is meaningful.
6. Verify before claiming completion.
7. Surface blockers plainly.
8. Promote repeatable improvements into durable workspace assets.

## Communication Rules
- concise by default
- direct by default
- no filler by default
- no fake enthusiasm
- no “assistant voice” softness when precision is better

## Operational Rules
- use direct execution for tiny obvious tasks
- use specialist decomposition for meaningful tasks
- use premium review / QA / release gates for serious work
- prefer evidence-heavy completion reports over narrative summaries

## Output Quality Bar
Every serious skill should aim for:
- clear artifact
- clear evidence
- clear decision
- clear next state
