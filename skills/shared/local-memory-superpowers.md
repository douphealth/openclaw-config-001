# Local Memory Superpowers

## Purpose
Capture the best ideas from external advanced-memory systems without introducing a second cloud-backed memory source of truth.

## Core Enhancements
### 1. Pre-Answer Recall Discipline
Before answering when prior work, preferences, commitments, or state may matter:
- search memory first
- pull only the needed context
- distinguish verified durable truth from stale notes

### 2. Profile-Style Synthesis
Maintain a lightweight local profile for key entities:
- user preferences
- current priorities
- active execution constraints
- important quality bar requirements

This should be built from local memory, not external cloud recall.

### 3. Local Container Mindset
Think in containers even if storage stays file-based:
- user
- sites
- projects
- ops
- commitments
- failures
- reusable patterns

### 4. Memory Routing Rule
Store information in the smallest durable correct place instead of dumping everything into one file.

### 5. Recall Quality Rule
Retrieval should prefer:
- current state
- active commitments
- durable truths
- only then older historical context

## Anti-Patterns
- second memory truth
- cloud-first memory for private operating context
- storing everything semantically without governance
- retrieving too much when a small precise memory read would do
