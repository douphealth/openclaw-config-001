# Perfect Post Structure Architecture

## Universal Post Structure Framework
All post types use this structural spine; only core-content internals vary by type.

## The Complete Post DNA
1. **SEO head (invisible)**
   - Title tag (50–65 chars)
   - Meta description (145–160 chars)
   - Canonical URL
   - Schema JSON-LD
   - Open Graph / Twitter cards
2. **Hero zone (0–5% scroll)**
   - H1 (intent-aligned; not identical to title)
   - Byline (author/date/read time/updated)
   - Hero image when appropriate
3. **Hook (5–10%, 100–150 words)**
   - Mirror reader context
   - Establish credibility
   - Promise specific value
   - Create forward momentum
   - Include 1 internal hub link
4. **Answer-first block (10–12%, 40–70 words)**
   - Direct answer, core entity + support entities
   - One source-backed fact
   - Self-contained extraction target
5. **Quick verdict/summary (12–15%)**
   - Commercial: quick picks table
   - How-to: step summary
   - Explainer: key takeaways box
6. **Table of contents (15–17%)**
   - Required if post > 2,000 words
7. **Context/methodology (17–22%)**
   - audience fit, prerequisites, scope, methods
   - 1–2 supporting internal links
8. **Core content (22–75%)**
   - H2/H3 sections with evidence + examples + visuals
   - 1–2 contextual internal links per section
   - one visual module every 500–800 words
9. **Synthesis/verdict (75–82%)**
   - context-dependent recommendation + limitations
   - include most relevant money-page link if natural
10. **Action section (82–88%)**
    - specific next steps + decision framework + CTA (if relevant)
    - 2–3 internal links to next-step content
11. **FAQ (88–95%)**
    - 6–10 entity-rich Q&A
    - 40–80 words per answer
    - NEW info (not body repetition)
    - FAQ schema only when FAQ is visible
12. **Related reading (95–97%)**
    - 3–5 curated internal links with short descriptors
13. **Trust footer (97–100%)**
    - editorial method, disclosure, last reviewed, credentials

## Scroll-Depth Engagement Map
| Zone | Reader state | Goal | Key elements |
|---|---|---|---|
| 0–15% | Evaluate | relevance + immediate reward | Hook, answer-first, quick summary |
| 15–25% | Commit | trust + expectation setting | TOC, methodology |
| 25–50% | Learn | core value delivery | best sections, visuals, evidence |
| 50–75% | Deepen | specificity escalation | advanced sections, case examples |
| 75–90% | Decide | action confidence | verdict, steps, CTA, FAQ |
| 90–100% | Complete | retention + trust | related reading, trust footer |

## Internal Link Audit (Before Publish)
- [ ] Hub/pillar page linked at least once (intro or first section)
- [ ] Total internal links within range for post length
- [ ] No section longer than 500 words without an internal link
- [ ] No generic anchors (click here/read more/learn more)
- [ ] No exact-match anchor repeated >1x per post
- [ ] Anchor text varied across post
- [ ] Each link provides reader value (not forced)
- [ ] No more than 3 links in one paragraph
- [ ] Same target URL not linked more than twice
- [ ] At least 2–3 links in top 30% of content
- [ ] All targets return HTTP 200
- [ ] Money-page links have natural context
- [ ] Related reading section has 3–5 links
- [ ] 3–5 inbound links from existing posts queued (orphan prevention)

## Post Assembly Master Checklist (Pre-Publish Quality Gate)
### Structure + UX
- [ ] Universal Post DNA sections present and correctly ordered
- [ ] Scroll-depth map respected (hook/reward/depth/decision/close)
- [ ] Visual rhythm rule met (1 visual per 400–600 words)
- [ ] Mobile 375px check passed (<=3 consecutive paragraphs max)

### SEO + AEO
- [ ] Title/meta/H1 aligned to intent
- [ ] Answer-first block present and extractable
- [ ] FAQ visible + schema eligibility validated
- [ ] Canonical + schema + social metadata valid

### Links + Conversion
- [ ] Internal link audit passed
- [ ] Conversion elements feel editorially natural
- [ ] Disclosure present near first affiliate link when applicable

### Trust + Evidence
- [ ] Claims dated + sourced or explicitly qualified
- [ ] Methodology visible
- [ ] Last-reviewed date and author/reviewer signals present
