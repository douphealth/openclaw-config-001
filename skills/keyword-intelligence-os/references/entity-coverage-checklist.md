# Entity Coverage Checklist (Strict)

For each URL, pass all checks:

1. Core entity definition present in first 120 words.
2. At least 3 supporting entities explained with role/context.
3. At least 2 contextual entities (audience, constraints, scenarios).
4. Synonyms/aliases included when search language varies.
5. Entity mentions distributed across title/H1/H2+FAQ (not clustered unnaturally).
6. Any tool/brand entity included only when genuinely relevant.
7. FAQ contains entity-based user questions.
8. No fabricated facts tied to entities.

Failing any check => status partial, not done.
