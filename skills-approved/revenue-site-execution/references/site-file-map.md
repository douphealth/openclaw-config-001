# Site File Map

For a managed site, check these folders:
- `ops/sites/` → business role, money path, worker mix, highest-value tasks
- `ops/constraints/` → operational traps and caveats
- `ops/launch-packs/` → launch / verification priorities
- `memory/sites/` → historical context and site notes

Use the site file as the primary routing source. Use memory for context, not as the first routing document.
