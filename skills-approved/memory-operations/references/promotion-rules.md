# Promotion Rules

## Daily memory
Use for:
- chronology
- state changes
- decisions
- blockers
- completed work worth remembering

## Entity memory
Use for:
- current site/project/person/system state
- active blockers
- access facts
- operational continuity
- durable next actions

## MEMORY.md
Use for:
- stable preferences
- durable truths
- recurring patterns
- long-term lessons

## Rule
Promote into entity memory first when the fact belongs to a real subject. Promote into `MEMORY.md` only when it should shape many future sessions.
