# Page Frameworks

## Homepage
Use when the page must orient multiple visitor types quickly.

Typical order:
- broad value proposition
- trust signal
- key benefits
- how it works
- proof
- paths for different visitor intents
- CTA

## Landing page
Use when one traffic source or one offer dominates.

Typical order:
- promise
- proof
- pain / stakes
- solution
- benefits
- objection handling
- CTA

## Pricing page
Use when visitors are close to evaluation or purchase.

Focus on:
- plan differences
- who each plan is for
- risk reduction
- FAQ
- buying confidence

## Headline patterns
- Achieve [outcome] without [pain]
- The [category] for [audience]
- Stop [bad state]. Start [better state].
- [Outcome], with less [friction]

## CTA patterns
Use strong action + clear outcome:
- Start free trial
- Get the checklist
- See pricing
- Book the strategy call
- Get my audit
