# Conversion Copywriting

## Purpose
Write clear, persuasive page copy that moves a specific audience toward a specific action.

## Use this when
Use this skill when the user needs:
- new marketing copy for a page or offer
- stronger headlines, subheads, CTAs, or section copy
- clearer value propositions
- tighter page messaging before design or CRO work



## Do this
1. Identify the page type and the single primary conversion action.
2. Establish audience, pain point, desired outcome, objections, and proof.
3. Mirror customer language where possible.
4. Write for clarity first, persuasion second, cleverness third.
5. Keep one main idea per section.
6. Connect features to outcomes and outcomes to business or personal value.
7. Provide 2-3 headline / CTA options when choice matters.

## Core rules
- Clarity beats cleverness.
- Specificity beats vagueness.
- Benefits beat feature lists.
- Honest proof beats hype.
- One page should usually push one primary action.
- Never invent numbers, testimonials, or claims.
- Do not produce safe generic SaaS sludge; if the copy could fit 50 other products, it is not done.
- For blog-style commercial content, optimize for human usefulness first, then SEO, GEO, and AEO structure without sounding machine-written.
- Rich on-page UX elements should support comprehension and scanning: summary boxes, comparison tables, checklists, FAQs, callouts, pull quotes, visual separators, media blocks, and well-placed embeds.
- For major editorial pages, aim to satisfy the full reader journey in one article: definition, workflow, tools, mistakes, examples, FAQs, next steps, and references.
- The strongest long-form posts also include implementation detail, scenario examples, useful contrasts, and a clear action path instead of stopping at abstract explanation.
- Run a semantic gap pass before finalizing: identify missing entities, missing subtopics, and missing reader questions compared with strong competing pages or obvious topic expectations.
- Use one optimized overhead title only; do not duplicate the main title inside the body as a second H1.
- Always create a short, clean, keyword-relevant slug.
- Always define a focus keyphrase for major editorial assets.
- Always create an SEO meta title (target <=59 chars) and meta description (target <=150 chars) and populate them through the active WordPress SEO/plugin path when the surface supports it.
- Internal links must be contextual, rich-anchor, verified against live 200-status targets whenever practical, and distributed naturally across the article rather than clumped into one section.
- FAQ sections should use visually distinct, engaging HTML presentation when the publishing surface allows it, so they stand out as a high-value skimmable section instead of reading like an afterthought.
- Major editorial pages should feel designed, not dumped: use beautiful HTML structure that improves readability and engagement without becoming gimmicky.

## Default page flow
Use or adapt this structure:
- headline
- subheadline
- primary CTA
- proof / trust signal
- problem or missed-opportunity framing
- solution / benefits
- how it works
- objections / FAQ
- final CTA

## Output
Default output should include:
- page goal
- audience summary
- primary message
- section-by-section copy
- 2-3 headline options
- 2-3 CTA options
- short rationale for key decisions when useful

## Checks and common mistakes
- Do not write generic claims like "streamline your workflow" without specifics.
- Do not let one section pitch multiple unrelated ideas.
- Do not bury the CTA behind unnecessary scene-setting.
- Do not use corporate sludge when plain English works.
- Do not confuse brand voice with vagueness or filler.
