# Doc Type Router

## Tutorial
Use when the reader is learning by doing.
Output should be sequential and beginner-safe.

## How-to
Use when the reader already knows the basics and wants to complete a task.
Output should be concise and task-oriented.

## Reference
Use when the reader needs facts, options, parameters, or exact behavior.
Output should be structured for scanning.

## Explanation
Use when the reader needs conceptual understanding or tradeoffs.
Output should answer why, not just how.
